🔎 漏洞明细

#漏洞名称等级状态修复措施
1明文密码存储🔴 高危✅ 已修复使用 generate_password_hash() 进行 bcrypt 哈希存储，登录时通过 check_password_hash() 比对。
2页面泄露明文密码🔴 高危✅ 已修复新增 get_user_info() 函数，返回模板前主动移除 password 字段，删除 index.html 中的密码行渲染。
3HTML 注释泄露默认账号🔴 高危✅ 已修复删除 login.html 中暴露默认管理员账号密码的 HTML 调试注释。
4Secret Key 硬编码🟡 中危✅ 已修复从硬编码改为读取环境变量 SECRET_KEY，未设置时自动生成 64 字符随机字符串。
5缺乏 CSRF 防护🟡 中危✅ 已修复引入 flask_wtf.CSRFProtect 中间件，表单添加 csrf_token 隐藏字段，缺失 Token 请求返回 HTTP 400。
6无权限校验🟡 中危✅ 已修复重构为 get_user_info() 函数，基于 session 取值，不依赖用户传入参数。
7Debug 模式始终开启🟢 低危✅ 已修复改为环境变量 FLASK_DEBUG 控制，生产环境默认关闭。
8登录接口无频率限制🟡 中危✅ 已修复引入 flask_limiter，登录 POST 限制为每分钟最多 5 次，超出返回 HTTP 429。
