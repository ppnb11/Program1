## 🔎 漏洞明细

### 1. 🔴 SQL注入 — 搜索接口
- **修复**: f-string 拼接 SQL 改为**参数化查询**（`?` 占位符），UNION 注入、盲注均被拦截。

### 2. 🔴 SQL注入 — 注册接口
- **修复**: 注册 SQL 改为参数化查询，用户输入不再拼入 SQL 语句。

### 3. 🔴 数据库明文密码存储
- **修复**: 初始化及注册均使用 `generate_password_hash()` 哈希存储。

### 4. 🔴 登录/注册数据不同步
- **修复**: 登录验证从硬编码 USERS 字典改为统一查询 SQLite 数据库。

### 5. 🔴 UNION注入提取密码
- **修复**: 参数化查询 + 搜索仅选取 id, username, email, phone，密码字段不返回模板。

### 6. 🟡 注册无频率限制
- **修复**: 注册接口添加 `@limiter.limit("10 per minute")`。

### 7. 🟡 无密码强度校验
- **修复**: 添加 `validate_password()`，密码长度至少 6 位。

### 8. 🟡 无输入校验
- **修复**: 用户名 3-20 位字母数字中文、邮箱格式校验、手机号格式校验。

### 9. 🟡 错误信息泄露数据库结构
- **修复**: 捕获 SQL 异常返回通用提示。

### 10. 🟡 搜索返回含密码字段
- **修复**: SELECT * 改为 SELECT id, username, email, phone。
