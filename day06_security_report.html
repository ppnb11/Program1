本次审计针对新增的动态页面加载功能（/page 路由）进行深度安全检测，发现路径穿越导致任意文件读取漏洞，可泄露 /etc/shadow、~/.ssh/id_rsa、应用源码等全部服务器敏感文件。所有漏洞均已完成修复并通过 11 项验证测试。

📋 涉及功能范围

  • 动态页面加载功能（/page 路由，GET 请求）
  • 文件路径拼装与读取逻辑
  • pages/ 目录下的静态页面文件
  • SSH 密钥文件安全（~/.ssh/ 目录）
  • 服务器系统敏感文件（/etc/passwd、/etc/shadow）

🔎 漏洞明细

#漏洞名称等级状态修复措施
1 FL-01 路径穿越 · 任意文件读取 (LFI)
/page 路由未校验用户输入，可读取服务器任意文件		 🔴 严重 ✅ 已修复 添加三层防护机制：① 入口过滤层：禁止 .. 和绝对路径；② 路径规范层：os.path.realpath() 解析规范化路径；③ 目录限制层：startswith(pages_dir) 确保文件在 pages/ 目录内。使用 os.path.isfile() 防止目录/符号链接绕过。
2 FL-02 SSH 私钥泄露 · 系统完全沦陷
通过 FL-01 读取 ~/.ssh/id_rsa，获取服务器 root 权限		 🔴 严重 ✅ 已修复 此漏洞是 FL-01 的直接利用方式。修复 FL-01 后，/root/.ssh/id_rsa、github_deploy、authorized_keys 等全部 SSH 文件均无法访问。建议事后更换密钥对并部署 fail2ban 等防护。

🎯 FL-02 攻击链详解

FL-02 并非独立漏洞，而是 FL-01 路径穿越的最高危利用方式——将"信息泄露"升级为"服务器完全沦陷"：

步骤操作Payload / 命令
① 发现 SSH 私钥 /page?name=/root/.ssh/id_rsa → 页面回显完整 RSA 私钥
② 保存私钥文件 复制私钥内容 → 保存为 stolen_key → chmod 600 stolen_key
③ SSH 免密登录 ssh -i stolen_key root@<IP> → 无需密码获取 root shell
④ 完全控制系统 窃取数据 / 篡改配置 / 部署后门 / 横向移动 / 清除日志
⚠ 极高风险：服务器上实际存在 5 个 SSH 相关文件（id_rsa、id_rsa.pub、github_deploy、authorized_keys、known_hosts），均可通过 FL-01 路径穿越一次性全部窃取。

📝 关键代码变更

🛡️ 入口过滤 + realpath 规范化 + 目录前缀校验

- @app.route("/page")
- def page():
-     name = request.args.get("name", "")
-     file_path = os.path.join("pages", name)   # 直接拼接，无校验
-     if os.path.exists(file_path):
-         with open(file_path, "r", ...) as f:   # 可读取 /etc/shadow
-             page_content = f.read()

+ @app.route("/page")
+ def page():
+     name = request.args.get("name", "")
+     # ✅ 第一层：入口过滤 — 禁止 ../ 和绝对路径
+     if ".." in name or name.startswith("/") or name.startswith("\\"):
+         page_content = "页面不存在"
+         return render_template("index.html", ...)
+     # ✅ 第二层：路径规范化 — realpath 消除符号链接
+     pages_dir = os.path.realpath("pages")
+     file_path = os.path.realpath(os.path.join("pages", name))
+     # ✅ 第三层：目录前缀校验 — 确保文件在 pages/ 内
+     if not file_path.startswith(pages_dir):
+         page_content = "页面不存在"
+         return render_template("index.html", ...)
+     # ✅ 文件类型校验 — 防止目录/符号链接绕过
+     if os.path.exists(file_path) and os.path.isfile(file_path):
+         with open(file_path, "r", ...) as f:
+             page_content = f.read()

💡 安全建议

🔒 额外安全措施建议：

1. 更换 SSH 密钥对：立即使用 ssh-keygen -t ed25519 重新生成密钥对，移除旧的公钥授权
2. 加固 SSH 配置：禁止 root SSH 登录（PermitRootLogin no），限制 AllowUsers
3. 部署 fail2ban：配置 SSH 暴力破解防护，自动封禁异常 IP
4. 页面名称白名单：限制 /page 路由仅允许预定义的页面名称
5. 请求日志审计：记录所有 /page 请求的完整 URL 参数，便于事后追溯
6. WAF 防护：部署 Web 应用防火墙拦截路径穿越攻击特征
