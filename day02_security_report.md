## 🔎 漏洞明细

### 1. 🔴 明文密码存储
- **修复**: 使用 `generate_password_hash()` 进行 bcrypt 哈希存储，登录时通过 `check_password_hash()` 比对。

### 2. 🔴 页面泄露明文密码
- **修复**: 新增 `get_user_info()` 函数，返回模板前主动移除 password 字段，删除 index.html 密码行渲染。

### 3. 🔴 HTML 注释泄露默认账号
- **修复**: 删除 login.html 中暴露默认管理员账号密码的 HTML 调试注释。

### 4. 🟡 Secret Key 硬编码
- **修复**: 从硬编码改为读取环境变量 `SECRET_KEY`，未设置时自动生成 64 字符随机字符串。

### 5. 🟡 缺乏 CSRF 防护
- **修复**: 引入 `flask_wtf.CSRFProtect` 中间件，表单添加 csrf_token 隐藏字段。

### 6. 🟡 无权限校验
- **修复**: 重构为 `get_user_info()` 函数，基于 session 取值，不依赖用户传入参数。

### 7. 🟢 Debug 模式始终开启
- **修复**: 改为环境变量 `FLASK_DEBUG` 控制，生产环境默认关闭。

### 8. 🟡 登录接口无频率限制
- **修复**: 引入 `flask_limiter`，登录 POST 限制每分钟最多 5 次，超出返回 HTTP 429。
